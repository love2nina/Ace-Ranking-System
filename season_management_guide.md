# 🎾 ACE 랭킹 시스템: 차기 시즌 및 누적 데이터 관리 가이드 (v1.1)

이 문서는 하반기 신규 시즌이 시작될 때 **실질 실력(MMR)은 누적 보존**하고, **시즌 랭킹(Rating)은 매번 새롭게 시작**하여 시스템의 공정성과 재미를 동시에 확보하기 위한 설계 가이드입니다.

---

## 1. 핵심 철학 (Core Strategy)

*   **MMR (누적 실력 점수)**: **절대 초기화하지 않습니다.** 수년간 쌓인 풍부한 매치 데이터를 바탕으로 그 사람의 진짜 테니스 실력을 정의하며, 매칭 밸런스를 맞추는 핵심 엔진 역할을 합니다.
*   **Rating (시즌 랭킹 점수)**: **시즌마다 1500점에서 새롭게 시작**합니다. "이번 시즌에 누가 제일 열심히 활동하고 좋은 성적을 냈는가"를 보여주는 지표이며, 신규 회원과 기존 회원 모두에게 새로운 동기를 부여합니다.
*   **Badge Hall & Insights**: '누적 결과'와 '시즌 결과'를 선택해서 볼 수 있게 하여 명예와 분석의 재미를 동시에 제공합니다.

---

## 2. 추천 데이터 구조 (Schema)

각 선수(`member`) 객체는 다음과 같이 두 가지 성격의 점수를 분리하여 관리합니다.

```javascript
{
    "id": "player_unique_id",
    "name": "홍길동",
    
    // [현행] 시즌 랭킹 (시즌마다 1500으로 Reset)
    "rating": 1500, 
    
    // [보안/전략] 실질 누적 실력 (Carry-over)
    "mmr": 1640,
    "baseMmr": 1640, // 시즌 시작 시점의 mmr을 보관 (시즌 변동분 계산용)
    
    // [신규] 역대 누적 합계 요약 (All-Time)
    "cumulativeStats": {
        "totalMatches": 230,
        "totalWins": 115,
        "totalBagels": 12
    },
    
    "prevSeasonStats": { ... } // 상대방별 누적 전적
}
```

## 3. 기능 구현 가이드

### STEP 1: 시즌 전환 프로세스 (`firebase-api.js`)
1. **MMR 보존**: 기존 시즌의 `mmr` 값을 새 시즌의 `mmr`과 `baseMmr`로 그대로 복사합니다.
2. **Rating 초기화**: 모든 유저의 `rating`은 `1500`으로 고정하여 새 시즌을 시작합니다.
3. **통계 누적**: 현재 시즌의 승/패/베이글 달성 횟수를 `cumulativeStats` 필드에 합산(`+=`) 합니다.

### STEP 2: 랭킹 보드 및 명예의 전당 UI (`ui.js`)
1. **랭킹 보드 (메인)**: 항상 `rating`(1500점 시작)을 기준으로 정렬하여 이번 시즌의 활약상을 보여줍니다.
2. **명예의 전당 (Badge Hall)**:
    - **[🏆역대 누적]**: `cumulativeStats` 필드를 기준으로 배지를 계산합니다.
    - **[🌱이번 시즌]**: 현재 시즌의 `matchHistory` 배열만으로 배지를 계산합니다.
3. **개인 분석 (Insights)**:
    - 사용자가 선택한 모드에 따라 현재 데이터만 분석하거나, `prevSeasonStats`를 포함한 전체 역사를 분석합니다.

## 4. 추가 배지 및 통계 아이디어 (Future Features)

> **구현 시점**: 명예의 전당 / 개인 분석 기능 확장 시 함께 적용 예정

### 🏆 명예의 전당 추가 배지

| 배지명 | 조건 | 의미 |
|:---|:---|:---|
| **🤝 국민파트너** | 가장 많은 다양한 파트너와 경기한 사람 | 모두와 잘 어울리는 팀플레이어 |
| **👑 천적 제조기** | 특정 상대를 상대로 승률 90% 이상인 경우가 3명 이상인 사람 | 공포의 상대 |
| **🐣 킹슬레이어** | 자신보다 MMR 높은 상대를 이긴 횟수 최다 | 업셋 전문가 |
| **📅 출석왕** | 현재 연속 참석 회차 최다 기록 보유자 | 개근의 신 |

> **특히 추천**: `국민파트너`, `킹슬레이어`, `출석왕`은 커뮤니티 분위기를 살리고 하위권 회원의 이탈 방지에 효과적입니다.

### 🔍 개인 분석 추가 통계

| 통계명 | 설명 | 재미 포인트 |
|:---|:---|:---|
| **롤러코스터 지수** | 매 경기 점수 변동폭(Rating Volatility) | "안정형 vs 극단형" 비교 |
| **파트너 궁합표** | 특정 파트너와 함께할 때의 승률 목록 | "같이하면 우리가 최강" 발견 |
| **나의 성장 스토리** | 첫 등록 MMR vs 현재 MMR 비교, 변화량 표시 | 성취감 부여 |
| **연속 참석 기록** | 현재 연속 참석 회차 수 실시간 표시 | 출석 도전 동기 부여 |
| **역대 고점 MMR** | 내가 기록한 최고 MMR 및 달성 시즌 | "나 한때 잘했어" 자랑 |

### 📦 `cumulativeStats` 확장 스키마

위 배지/통계를 지원하기 위해 `cumulativeStats` 필드를 다음과 같이 확장합니다.

```javascript
"cumulativeStats": {
    "totalMatches": 230,
    "totalWins": 115,
    "totalBagels": 12,
    "kingsSlayerCount": 8,        // MMR 높은 상대 이긴 횟수
    "uniquePartners": 12,          // 함께 팀을 이뤄본 파트너 수
    "consecutiveAttendance": 5,    // 현재 연속 참석 회차
    "maxConsecutiveAttendance": 9, // 역대 최고 연속 참석 기록
    "extremeMatchCount": 7,        // 6:0 또는 0:6 경기 횟수
    "peakMmr": 1680                // 역대 최고 MMR
}
```
